## Task 1 — H1: Filter archived firearms from session `findFirst` fallback

**File:** `src/app/api/range/sessions/route.ts`

When no `firearmId` is passed, the POST handler picks the oldest firearm. Without `archivedAt: null`, this could return an archived gun.

- [ ] **Step 1: Edit the findFirst fallback (line 171)**

  Change:
  ```ts
  : (await prisma.firearm.findFirst({ select: { id: true }, orderBy: { createdAt: "asc" } }))?.id;
  ```
  To:
  ```ts
  : (await prisma.firearm.findFirst({ where: { archivedAt: null }, select: { id: true }, orderBy: { createdAt: "asc" } }))?.id;
  ```

- [ ] **Step 2: Also guard the explicit firearmId path (line 179)**

  The `findUnique` on line 179 validates a user-supplied `firearmId`. It should also reject archived firearms. Change:
  ```ts
  const firearm = await prisma.firearm.findUnique({
    where: { id: resolvedFirearmId },
    select: { id: true, caliber: true, compatibleCalibers: true },
  });
  ```
  To:
  ```ts
  const firearm = await prisma.firearm.findUnique({
    where: { id: resolvedFirearmId, archivedAt: null },
    select: { id: true, caliber: true, compatibleCalibers: true },
  });
  ```
  The existing `if (!firearm)` check below already returns a 404, so this reuses that path correctly.
