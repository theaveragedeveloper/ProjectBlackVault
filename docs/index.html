Deployment

Production Deployment Walkthrough

Deploy with Docker Compose using built-in install scripts. This path is production-oriented and avoids dev-mode terminal setup.

Terminal
1
Install prerequisites
Docker Desktop (Windows/macOS) Docker Engine + Compose plugin (Linux)
2
Clone and run first-time installer
git clone https://github.com/theaveragedeveloper/ProjectBlackVault.git cd ProjectBlackVault # Linux/macOS chmod +x install.sh && ./install.sh # Windows install.bat
3
Verify deployment health
docker compose --env-file .blackvault.env ps docker compose --env-file .blackvault.env logs -f http://localhost:3000/api/health
4
Access from other devices
# Find host IP, then open from phone/laptop: http://YOUR-HOST-IP:3000
5
Day-2 operations
# Start / stop / logs docker compose --env-file .blackvault.env up -d docker compose --env-file .blackvault.env down docker compose --env-file .blackvault.env logs -f # Update ./update.sh (Linux/macOS) update.bat (Windows)
6
Backup and recovery essentials
Back up .blackvault.env and your DATA_DIR. Do NOT lose VAULT_ENCRYPTION_KEY. Use Settings → Secure Backup for encrypted exports.

Desktop launcher: Download Installer (falls back to all releases if platform build is unavailable)

Advanced channels: winget install --id TheAverageDeveloper.ProjectBlackVault · brew install --cask theaveragedeveloper/tap/projectblackvault
